GOAL: replace the multi-layer slot pin stack (shadow/border/fill/rank × 30 slots = 120 layers + moveLayer z-order + nativeLodZ reassignment + promote-await + crossfade coupling) with ONE symbol per pin: `icon-image` = pre-baked per-bucket sprite, `text-field` = rank glyph, `symbol-z-order: viewport-y` for native screen-Y stacking. Makes pin source FULLY RESIDENT → LOD = pure feature-state opacity → no source mutation on promote → no await → clean crossfade → and enables Google-style "everything in the search is available, viewport+collision reveals it". See map-lod-target-plan.md, map-lod-demotion-root-cause.md.

WHY THIS IS THE IDEAL (research-backed 2026-05-31): Mapbox render time scales with LAYER count (docs: combine layers to reduce render time; 1000+ symbol-image layers drop frames) but is built for many FEATURES in ONE layer (GridIndex viewport-coords collision, CrossTileSymbolIndex). So resident-slots (keep stacks) does NOT scale — it's layers. Single-symbol scales to thousands of features in one layer = how Google does it. SDF is single-color so can't do our multicolor pin; the fix is pre-baked raster sprites at device DPI (scale/pixelRatio=3, icon-size=1) — sharpness was only ever lost from 1x-upscaling, not from raster per se.

BUCKETS (confirmed w/ user): 4 decade buckets over the 60-100 display range (displayMin:60 displayMax:99.9 from public-crave-score.service.ts), remapped across the FULL palette so color carries signal (today t=1-score/100 only ever uses green→yellow third; orange/red never render). Colors (via quality-color.ts interpolation, bucket center score):
  60s(64.5)=rgb(255,128,79) red-orange | 70s(74.5)=rgb(255,168,80) amber | 80s(84.5)=rgb(212,201,100) yellow-green | 90s(95)=rgb(59,196,123) green
Rationale: learnable score↔color association ("90s=green=best"); 8-10 buckets indistinguishable on a small pin + destroy the intuition. If live range is actually 70-100 → drop to 3 buckets (70s/80s/90s).

GEOMETRY GUARANTEE (honored): fill centered in base BODY, shifted UP off the tip. Source art: pin.png=480x480 (full body silhouette incl tip, white alpha mask, 5x of 96 logical), pin-fill.png=400x360 (inner pill, white alpha mask, 5x of 80x72). Live offsets: PIN_FILL_TOP_OFFSET=(96-72)/2 + (-4 VERTICAL_BIAS)=8 logical; PIN_FILL_LEFT_OFFSET=(96-80)/2=8 logical. Generator reuses these EXACT formulas → pixel-identical placement.
